COMPLETE DATABASE SCHEMA (Prisma):

1. Cohort Model:
   - id, name, description
   - startDate, endDate
   - applicationDeadline
   - status (DRAFT, RECRUITING, MATCHING, ACTIVE, COMPLETED, ARCHIVED)
   - maxMentors, maxMentees
   - isPublic (for public applications)
   - settings (JSON - for cohort-specific configurations)
   - createdById (Admin who created)
   - createdAt, updatedAt

2. Track Model (the 5 SONSIEL tracks):
   - id, name, slug, description
   - icon (for UI display)
   - color (hex code for UI)
   - isActive
   - sortOrder
   
   Seed with: Scientist, Innovator, Entrepreneur, Intrapreneur, Leader

3. CohortTrack (junction table):
   - cohortId, trackId
   - isActive for this cohort
   - mentorCapacity
   - menteeCapacity

4. CohortMembership:
   - id
   - cohortId, userId
   - role (MENTOR or MENTEE in this cohort)
   - trackId (which track they're in)
   - applicationStatus (PENDING, APPROVED, REJECTED, WAITLISTED)
   - applicationData (JSON - stores survey responses)
   - matchStatus (UNMATCHED, MATCHED, REMATCHED)
   - joinedAt
   - completedAt
   - certificateIssuedAt
   - notes (admin notes)

5. MentorshipMatch:
   - id
   - cohortId
   - mentorMembershipId
   - menteeMembershipId
   - trackId
   - status (PROPOSED, ACTIVE, PAUSED, COMPLETED, TERMINATED)
   - matchScore (from algorithm)
   - matchReason (JSON - explains why matched)
   - matchedAt, matchedById (admin who approved)
   - startDate, expectedEndDate, actualEndDate
   - terminationReason
   - mentorFeedback, menteeFeedback (JSON)
   - createdAt, updatedAt

6. MeetingLog:
   - id, matchId
   - scheduledAt, occurredAt
   - duration (minutes)
   - meetingType (VIRTUAL, IN_PERSON, PHONE)
   - agenda, notes, actionItems (rich text)
   - mentorAttended, menteeAttended
   - nextMeetingDate
   - createdById

7. Goal:
   - id, matchId
   - title, description
   - category (SHORT_TERM, LONG_TERM, MILESTONE)
   - targetDate
   - status (NOT_STARTED, IN_PROGRESS, COMPLETED, ABANDONED)
   - progress (0-100)
   - createdById, assignedToId
   - completedAt
   - reflectionNotes

8. Task:
   - id
   - title, description
   - assignedById (Admin or Mentor)
   - assignedToId (Mentor or Mentee)
   - matchId (optional - if task is match-specific)
   - cohortId (optional - if cohort-wide task)
   - priority (LOW, MEDIUM, HIGH, URGENT)
   - status (TODO, IN_PROGRESS, BLOCKED, COMPLETED, CANCELLED)
   - dueDate
   - reminderDate
   - completedAt
   - attachments (relation to Document)
   - tags (array)
   - parentTaskId (for subtasks)

9. Document Model:
   - id, name, description
   - fileUrl, fileType, fileSize
   - category (TEMPLATE, AGREEMENT, RESOURCE, SUBMISSION, OTHER)
   - visibility (PUBLIC, COHORT, TRACK, MATCH, PRIVATE)
   - uploadedById
   - cohortId (optional)
   - trackId (optional)
   - matchId (optional)
   - version
   - isTemplate
   - tags (array)
   - downloadCount
   - createdAt, updatedAt

10. DocumentAccess:
    - documentId, userId
    - accessType (VIEW, DOWNLOAD, EDIT)
    - grantedById
    - grantedAt, expiresAt
